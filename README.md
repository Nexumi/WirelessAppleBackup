# WirelessAppleBackup

## Run
Please have docker installed before starting.
```
chmod +x build extract move
./build
./extract
./move
```

## Setup
Run `sudo nano /usr/local/libexec/run-iphone-backup` and update `BACKUP_DIR` to select directory for backups.

Install screen: `sudo apt install screen`

## systemctl
```
sudo apt install usbmuxd -y && sudo apt purge usbmuxd -y && sudo apt autoremove -y
sudo nano /etc/systemd/system/usbmuxd2.service
```
```
[Unit]
Description=iOS USB and WiFi Muxer Daemon (usbmuxd2)
After=network.target avahi-daemon.service dbus.service
Requires=dbus.service
Wants=avahi-daemon.service

[Service]
Type=simple
Environment=LD_LIBRARY_PATH=/usr/local/lib/usbmuxd2
ExecStart=/usr/bin/stdbuf -oL /usr/local/sbin/usbmuxd2 --user usbmux --systemd
StandardOutput=journal
StandardError=journal
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```
```
sudo systemctl enable usbmuxd2
sudo systemctl start usbmuxd2
```

## Pairing your iDevice
- Physically connect your device to the system
- Run `idevicepair pair`
- Disconnect your device
- Run `ideviceinfo --network` to verify device can still be reached

> [!IMPORTANT]
> Use `--network` flag with idevice commands to tell them to reach the device wirelessly

> [!TIP]
> Use `--udid UDID` flag to target a specific device if multiple devices are paired with the system

> [!WARNING]
> If you run into permission issue with connecting the device (check with `sudo systemctl status usbmxud2`), do the following

## Permissions fix
```
sudo nano /etc/udev/rules.d/99-apple-iphone.rules
```
```
SUBSYSTEM=="usb", ATTR{idVendor}=="05ac", GROUP="plugdev", MODE="0660"
```
```
sudo udevadm control --reload-rules
sudo udevadm trigger
```

```
/usr/local/libexec/run-iphone-backup --udid "UDID" --dir "BACKUP NAME" --topic "TOPIC" --priority "PRIORITY"
```

## ntfy
> [!NOTE]
> Optional, but highly recommended for status tracking.

- Install [ntfy](https://apps.apple.com/us/app/ntfy/id1625396347) from the Apple App Store

THEN

- Use public API

OR

- Install ntfy onto the system. ntfy.yaml is provided for a docker instance. Feel free to configure or install with [any method](https://docs.ntfy.sh/install/).
- Recommended: Update the ntfy app to default to your custom server

FINALLY

- Run `sudo nano /usr/local/libexec/iphone-backup-helper` and update `NTFY_SERVER` to point to the chosen server.

## Uninstall
```
sudo systemctl stop usbmuxd2
sudo systemctl disable usbmuxd2
sudo rm /etc/systemd/system/usbmuxd2.service

sudo rm /usr/local/sbin/usbmuxd2
sudo rm /usr/local/bin/idevice_id
sudo rm /usr/local/bin/idevicebackup2
sudo rm /usr/local/bin/ideviceinfo
sudo rm /usr/local/bin/idevicepair

sudo rm /etc/ld.so.conf.d/usbmuxd2.conf
sudo ldconfig

sudo rm -rf /usr/local/lib/usbmuxd2

sudo rm /usr/local/libexec/iphone-backup-helper
sudo rm /usr/local/libexec/run-iphone-backup
```
